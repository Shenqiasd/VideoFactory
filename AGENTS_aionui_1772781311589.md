# AGENTS 全局协作规范

本文件适用于本仓所有 Coding Agent。发生冲突时，优先级：
1. 用户直接指令
2. 本文件
3. 各角色局部约定

## 1. 通用行为约束
- 仅在任务范围内修改文件；禁止顺手重构无关模块。
- 修改前先读取相关上下文，避免覆盖他人变更。
- 输出必须包含：改了什么、为什么、如何验证。
- 发现阻塞时立即标注，不做隐性假设。

## 2. 分支与变更粒度
- 一次任务一个主题，保持小而可审查。
- 提交信息需带任务编号：`[TASK-xxx]`。
- 变更必须可回滚，不提交不可逆脚本。

## 3. 代码质量底线
- 通过格式化与静态检查。
- 新增逻辑需配套测试（至少一个正向用例）。
- 禁止引入硬编码密钥、口令、令牌。
- 若降低覆盖率，必须说明理由与补救计划。

## 4. 审查与测试
- Reviewer 先报阻断问题，再给优化建议。
- Tester 必须记录测试范围、命令、结果、结论。
- 未通过 `governance/QUALITY_GATES.md` 的条目不得集成。

## 5. 交接格式
- 使用 `agents/templates/HANDOFF_NOTE.md`。
- 必填：上下文、变更清单、风险、待办、验证证据。

## 6. 安全与合规
- 不输出或提交敏感信息。
- 任何潜在破坏性操作需要明确批准。
- 第三方依赖新增需说明来源与必要性。
